# 新人一键接入提示词

## 使用方式

把下面整段复制给你的 Codex / Claude 客户端。你只需要把管理员给你的 `接入密钥` 和你的名字填进去。

AI 会自己从 GitHub 下载工作流文件，不需要你手动下载。

GitHub 仓库：

```text
https://github.com/Ruceshao/discord-workflow-onboarding
```

AI 会把文件保存到本地文件夹，例如：

```text
~/company-discord-workflow/
  新人一键接入提示词.md
  Codex客户端接入说明.md
  AI工作流手册.md
  AGENTS.discord-workflow.md
  setup-client.sh
  skills/discord-workflow/SKILL.md
```

```text
请帮我接入公司的 Discord 工作流。

请从这个 GitHub 仓库下载工作流文件：
https://github.com/Ruceshao/discord-workflow-onboarding

请把文件保存到这个本地文件夹：
~/company-discord-workflow

这是管理员给我的信息：
- API_URL：https://workflow.weebstrading.xyz
- 接入密钥：<填入管理员给你的 token>
- 我的名字：<填入你的名字>

请你自动完成配置，不要让我手动编辑文件。

具体要求：
1. 先 clone 或下载 GitHub 仓库到本地文件夹。
2. 读取本地文件夹里的 Markdown 文件，尤其是 `AGENTS.discord-workflow.md` 和 `Codex客户端接入说明.md`。
3. 检查我的电脑是否有 curl 和 bash。
4. 运行本地文件夹里的安装脚本：
   bash ~/company-discord-workflow/setup-client.sh --api-url "$API_URL" --token "$接入密钥" --submitter "$我的名字"
5. 安装完成后，读取 `~/.discord-workflow/AGENTS.discord-workflow.md`，确认规则已写入本机。
6. 如果我是 Codex 用户，确认 `~/.codex/skills/discord-workflow/SKILL.md` 已安装。
7. 帮我做一次 health check，但不要发布任何内容。
8. 以后当我说“发布到 Discord 工作流”时，你必须先给我看最终稿，等我明确确认后，再调用 `discord-workflow-publish`。
9. 以后当我要查看、回复、更新或关闭已有需求时，你必须先用 `discord-workflow-read REQ-xxxx` 读取上下文；如果我不知道编号，先用 `discord-workflow-list` 辅助定位。
10. 以后当我要补充进度、卡点、决策或说明时，你必须先给我看回复稿，等我确认后，再调用 `discord-workflow-reply`。
11. 在 Codex 里，优先使用 `discord-workflow` skill 处理这些请求。

注意：
- 不要把接入密钥写进项目仓库。
- 不要在我确认前发布。
- 不要只靠标题猜测要操作哪条需求，优先使用 `REQ-xxxx` 编号。
- 如果遇到错误，请直接修复或告诉我需要管理员处理什么。
```

## 管理员需要提供什么

管理员只需要给新人一项：

- `接入密钥`

新人不需要理解 `.env`、curl、JSON 或 Discord API。
